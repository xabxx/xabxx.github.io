The Future Us: Purpose, Features, Implementation

Long story short: Synthetic biology is my concentration/class focus in my major. A lot of people don't know what it is and I thought I would start a channel where people can get up-to-date synthetic biology feeds. The purpose of the this skill is to attract more bright minds into the field. Features include asking the definition of synthetic biology, getting synthetic biology feeds, and getting synthetic biology trivia. Feeds and trivia are pulled from here. I hand-pick news in the field and post them on this site, and the skill makes a request to the site each time the skill is called. Why did I name it "The Future Us"? Once upon a time we could not conceive the notion of automation and machines until a group of people realized the inconceivable dreams. The Code

The code is written in Javascript and has Node.js 6.10 runtime. Again I prefer not to go into details because the code pretty much explains itself (I guess for the most part). The alexa SDK API calls are the same as in the Alexa sample projects from their official github account. One thing that I want to highlight is the how I handled the asynchronous function - request. Below shows a snippet of the code that calls the asynchronous request.

Request call to the site where feeds and trivia are pulled from.

I included the call to registerHandlers and execute (line 87 and 88) in the callback function of the request call (line 72 - 89). This is ensure that there are at least one content in the arrays where trivia and news are stored and randomly picked respectively. Without implementing the code this way, the code at line 72 will make the request GET call to scrape the contents of the designated site, but it won't stop to wait for the call to finish - namely, the code will run pass line 72 and the arrays designed to store feeds and trivia will be empty. To prevent this from happening, I included the last two lines to code (registerHandlers & execute) inside the callback function. Therefore, unless there is a connection error, the contents in the arrays should reflect contents on the site.

I packaged the index.js file, package.json file, and the node_modules folder into a .zip file, and I uploaded it onto the lambda function I created. Two other modules the code depends on are request and cheerio.
